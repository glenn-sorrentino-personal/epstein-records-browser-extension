Public Records Overlay

See relevant names as you browse, instantly.

Epstein Records Indicator adds neutral, local-only badges next to exact name matches on any webpage. It uses a curated dataset with clear categories and a one-source sync workflow across Chrome, Firefox, and Safari. The project is fully open source and publicly auditable.

Badge categories

Every category is explicit, searchable, and editable from one canonical dataset.

epstein files epstein mentioned epstein collaborator epstein enemy

Matching runs locally in the browser. No dataset upload required.

How it works

The extension scans visible page text, normalizes names, and inserts category badges on exact matches. Each badge links to DOJ Epstein search results for quick source review.

1. Detect

Unicode-aware exact name matching across regular webpages.

2. Label

Color-coded badges for files, mentioned, collaborator, and enemy.

3. Verify

One-click jump to DOJ search context for additional review.

Local-only by design

This extension processes page text on-device only. It does not upload your browsing text, your local dataset, or your match history to any external server. The only outbound action is when you explicitly click a badge link to open DOJ search results in a new tab.

No cloud processing

Matching and normalization run entirely in the browser.

No dataset upload

Your local `data/*.txt` and `records.js` stay on your machine.

User-triggered outbound only

External navigation happens only when you click a badge.

One-source data workflow

Maintain data once in source-data/*. Sync script propagates to browser-specific folders, and CI validates or auto-fixes mirrored files.

Edit canonical files

source-data/records.js source-data/names.txt source-data/mentions.txt source-data/collaborator.txt source-data/enemy.txt

Sync + verify

bash scripts/sync-data.sh bash scripts/sync-data.sh --check

Install in minutes

Load unpacked for local testing, iterate on dataset changes, and refresh the extension.

Chrome

chrome://extensions -> Developer Mode -> Load unpacked -> select chrome/.

Firefox

about:debugging#/runtime/this-firefox -> Load Temporary Add-on -> firefox/manifest.json.

Safari

Open the Xcode project in safari/, build the app target, then enable extension in Safari settings.
